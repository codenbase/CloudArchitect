
### 1. 准备镜像包

- 编写 Dockerfile
	- 利用缓存加快 CI/CD 流程：先拷贝 go.mod 先下载依赖
	- 利用 -ldflags="-s -w" 减小 Go 二进制文件体积
	- 利用 -trimpath 移除隐私路径，避免暴露宿主机的用户名和目录结构
- 打包镜像
	- `docker build -t ghcr.io/codenbase/<镜像名>:<镜像版本> .`
	- 【可选】指定阿里云私有镜像库，加快速度
		- `--build-arg REGISTRY_PREFIX=registry.cn-hangzhou.aliyuncs.com/akuku/`
- 推送仓库（ghcr.io）
	- `docker push ghcr.io/codenbase/web-demo-backend:v1.0.0`
	- `docker push ghcr.io/codenbase/web-demo-frontend:v1.0.0`

### 2. 准备配置资源

- 创建命名空间 `web-demo`
- 创建 ConfigMap，存入 PostgreSQL 初始化建表 SQL 脚本
- 创建 Secret：存入敏感数据 `postgres`（数据库用户名、密码） 

永远不要为了适配 K8s 而去修改 Go 代码里的环境变量获取逻辑。用 K8s YAML 中的 `env.valueFrom` 提取 Secret，并通过 `$(VAR)` 语法直接在 YAML 里动态拼装成完整的 `DATABASE_URL`，实现真正的代码与环境解耦。

### 3. 部署 PostgreSQL 数据库

- 创建 PV，本地 docker 环境下的集群，直接使用 Local PV，需要配置**节点亲和性 (Node Affinity)**，把数据和 Pod 死死焊在同一台物理机上，防止 Pod 漂移后找不到数据。
	- Local PV 不会自动建目录。必须 SSH 到对应节点执行 `mkdir` 并赋予 `chmod 777` 权限。
- 创建 Headless Service，数据库集群绝对不能用普通 Service，必须指定 `clusterIP: None` 变为无头服务。
- 编写 StatefulSet YAML 文件

【可选】创建 StorageClass 连接外部存储。

### 4. 部署无状态服务（backend、frontend）

- 编写 Deployment 和 Service（ClusterIP） YAML 文件
- 使用节点的 `kubernetes.io/hostname` 进行 affinity 配置，防止业务 Pod 跑到控制平面抢资源。

### 6. 使用 Helm 一键部署。

- 通用模板 (Templates) + 环境配置 (Values.yaml) = 最终的 K8s YAML (Manifests)
- 一键部署/卸载
- 多环境管理
- 版本回滚
- .Release.Name 是你执行 `helm install` 时，紧跟在后面的那个名字。
- 动态修改 `values` 的“优雅”方式：
	- `--set` 是一把“快刀”，最适合 CI/CD 流水线。
	- 创建一个 `values-prod.yaml`，只写生产环境独有的配置，通过 `-f` 覆盖合并。
- 阿里云私有镜像仓库密码
	- 简单环境：提前手动执行 `kubectl create secret docker-registry ...` 
	- 企业级：引入 安全 Operator，把阿里云的密码存放在专门的“密码保险箱”（比如 HashiCorp Vault 或者阿里云 KMS）里。K8s 集群里运行的一个机器人，会自动去保险箱里读取密码，并在 `web-demo` 命名空间里**全自动地**生成那个 `aliyun-registry-secret`。
